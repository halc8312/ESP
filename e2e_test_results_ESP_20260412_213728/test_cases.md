## A. 認証フロー

### TC-E2E-001

| 項目 | 内容 |
|------|------|
| **ID** | TC-E2E-001 |
| **優先度** | Critical |
| **観点** | 正常系：ログイン成功 |
| **目的** | 正しい認証情報でログインし、ダッシュボードにリダイレクトされること |
| **前提条件** | ユーザー `testuser` / `password123` が登録済み |
| **手順** | 1. `GET /login` にアクセス<br>2. username=testuser, password=password123 を入力<br>3. フォーム送信 |
| **期待結果** | 302リダイレクト → `/` (ダッシュボード) |
| **追加確認** | セッションクッキーが HttpOnly=True, SameSite=Lax であること |
| **バグになりやすい理由** | リダイレクト先が変更された場合にセッション確立を確認せずに終わる |

### TC-E2E-002

| 項目 | 内容 |
|------|------|
| **ID** | TC-E2E-002 |
| **優先度** | Critical |
| **観点** | 異常系：誤パスワードでログイン失敗 |
| **目的** | 誤パスワードで認証が拒否され、エラーメッセージが表示されること |
| **前提条件** | ユーザー登録済み |
| **手順** | 1. `POST /login` に誤パスワードで送信 |
| **期待結果** | 200レスポンス、「ユーザー名またはパスワードが違います」表示 |
| **追加確認** | セッションが作成されていないこと |
| **バグになりやすい理由** | エラー表示のみでセッション作成チェックを忘れがち |

### TC-E2E-003

| 項目 | 内容 |
|------|------|
| **ID** | TC-E2E-003 |
| **優先度** | Critical |
| **観点** | 異常系：ログイン試行レート制限 |
| **目的** | 5回連続失敗後にレート制限メッセージが表示されること |
| **前提条件** | ログイン試行制限が 5回/15分 であること |
| **手順** | 1. 同一IPから誤パスワードで6回連続送信 |
| **期待結果** | 6回目に「ログイン試行回数が上限に達しました」表示 |
| **追加確認** | 6回目のレスポンスが200（ロックアウト画面）であること |
| **バグになりやすい理由** | カウンターがメモリ上のため、ワーカー再起動でリセットされる点を見落としがち |

### TC-E2E-004

| 項目 | 内容 |
|------|------|
| **ID** | TC-E2E-004 |
