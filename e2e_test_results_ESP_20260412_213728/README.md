## テストの目的

「表面的に動くことの確認」ではなく、**実運用で起こりうる不具合・仕様抜け・回帰・境界条件・例外系・非機能上の問題**を最大限あぶり出すこと。

具体的には以下を重点確認：

- 認証フロー（ログイン・登録・ログアウト・レート制限）の完全性
- 商品CRUD・価格ルール適用の正確性
- パブリックカタログのセキュリティ（内部情報漏洩チェック）
- 権限制御（未ログインユーザーの保護リソースアクセス）
- データ整合性（作成→更新→削除の一覧反映）
- 異常系・境界値（入力バリデーション、二重送信、セッション切れ）
