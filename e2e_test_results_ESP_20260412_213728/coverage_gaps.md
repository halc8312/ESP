## 残存リスク

### RR-01: selling_price 計算バグ（Critical）

- **BUG-E2E-001** が未修正の場合、パブリックカタログに誤価格が表示される
- 修正後の回帰テストが必要

### RR-02: 非同期処理の未テスト領域

- `gather_with_concurrency` は複数スクレイピングサイトの並行実行に使用
- テストなしでリリースすると、並行処理バグが本番で発覚するリスク

### RR-03: ログインレート制限のin-memory実装

- `_login_attempts` はプロセスメモリ上に保持
- ワーカー再起動・スケールアウトでリセット → レート制限が無効化
- 本番でワーカーが複数台ある場合は別プロセス間で状態共有されない

### RR-04: PostgreSQL vs SQLite の動作差異

- `String(100)` の超過時：SQLite は切り捨て vs PostgreSQL はエラー
- トランザクション分離レベルの差異
- `utc_now()` のタイムゾーン処理の差異

### RR-05: セレクタ変更によるスクレイピング失敗

- Mercari/Snkrdunk の HTML 構造変更でセレクタが無効化される
- `selector_config.py` および `services/selector_healer.py` で自動修復機能があるが、完全な保証はない

### RR-06: CSRF 保護のテスト非実施

- テストでは `WTF_CSRF_ENABLED=False` のため、本番で CSRF が実際に機能しているかの確認は手動のみ
- JavaScript の `X-CSRFToken` ヘッダー送信のテストがない

### RR-07: セッション管理の脆弱性

- セッション固定化攻撃（ログイン前後でセッションID が変わるか）の確認未実施
- `session_protection` レベルの設定確認が必要
