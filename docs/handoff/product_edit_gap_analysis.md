# 商品編集モック (PSA10) vs 現状コード ギャップ分析

対象モック: `~/attachments/4b81a64d-91c0-4c94-855b-8d08547bc25c/.html` (PSA10 ワンピースカード商品編集)
対象現状: `templates/product_detail.html` (PR #96 マージ後の状態 / `aa26325`)

結論: **大きく逸脱しています。** PR #96 で部分対応したのは「タグ pill UI」「画像ライトボックス」「個別の利益率/送料 (manual override)」の3点のみで、モック全体のレイアウト構造・情報設計とはほぼ別物です。

---

## レイアウト構造 (一番大きい差分)

| 観点 | モック | 現状 | 差分 |
|---|---|---|---|
| ページ構成 | サイドバー + 上部ヘッダー (タイトル/ステータス/保存) + メイン4列グリッド (`lg:grid-cols-4`、メイン3列・サイド1列) | サイドバー + Beginner-Friendly Flow ヒーロー + 単一カラムの `<details>` アコーディオン羅列 | **構造そのものが違う**。モックは「フラットなカード型セクション + 右サイドカラム」、現状は「縦積み accordion + 保存前チェック側カラム」 |
| セクション開閉 | 常時開きカード | `<details ... open>` accordion (折りたたみ可能) | accordion を畳むと項目を見落としやすい運用が、モックでは起きない |
| 保存 UI | ヘッダ右の `保存する`、モバイル時のみ画面下 sticky | デスクトップ下部の単独ボタン + モバイル sticky | ヘッダー保存ボタンが現状にはない |
| 状態バッジ | ヘッダー右に `下書き` (amber) を直に表示 | ヒーロー内のメタ列に `公開中` 等を表示 | 視認順序がずれる |

## メインカラム — セクション順と中身

モックの並び (上から):
1. 商品画像
2. 仕入設定 (自動取得) — 読み取り専用
3. 商品説明 (タイトル/説明 JP/EN を並列に並べた2列)
4. 販売設定 (販売価格 + 在庫 + 個別計算パラメータ)
5. SEO (URLハンドル/ページタイトル/メタディスクリプション)

現状の並び:
1. 基本設定 (所属ショップ + 公開ステータス + 仕入れ価格/状態の小タイル) ← モックには無い
2. 商品画像
3. 商品説明 (日本語/英語の縦積みエディタ)
4. 保存前チェック (右カラムにあるが現状では片側に固定で出ている)
5. バリエーション設定 (販売価格・在庫・SKU を含むテーブル + manual override) ← モックは販売設定として独立
6. 販売メモ・検索設定 (タグ/SEO/Vendor/Handle が混在) ← モックでは「分類」(右サイド) と「SEO」(メイン) に分離
7. 内部仕入れメモ ← モックには無い (代わりに上部の「仕入設定」が公開取得元情報を兼ねる)

→ **対応するセクションがほとんど無い**。「仕入設定」「販売設定」「SEO」の独立カードはモックにあって現状に無い構造。

## 1. 商品画像セクション

| 観点 | モック | 現状 |
|---|---|---|
| グリッド | `grid-cols-5 md:grid-cols-10 gap-2` (極小サムネ、最大10列) | 大きな縦長 `.product-image-card` を概ね1〜3列で配置 |
| カード内容 | `aspect-square` の画像のみ。番号ラベル/URL文字列なし | 画像 + `1枚目` ラベル + `≡ 並べ替え` + URL文字列 + 白抜きステータス |
| アクションボタン | 24px 円形アイコン (white icon on blue/red)、hover 時のみ `opacity:0→1` でカード右上に重なる | 文字ボタン `白抜き` `✕` がカード上部に常時表示 |
| ライトボックス | カード全体クリックで開く。`#lightbox` は body 直下、`event.stopPropagation` で action ボタンの誤発火回避 | サムネクリックで開く。実装方針はほぼ同じだが UI トリガー領域が画像のみ (周辺 padding は別) |
| 並べ替え | Sortable.js (`#imageGrid`) | 既存の独自実装 (`image-sort-handle`) |
| 追加 UI | グリッド末尾の `drop-zone` (1セル分の点線枠) | 別ブロック (URL欄 + ファイル選択欄) を下にぶら下げ |
| `URLから追加` | ヘッダー右の `text-slate-600 bg-slate-100` ボタン | ツールバーに同名ボタン (役割は同じ) |

→ サムネサイズ・密度が桁違い。モックは10列の "コンタクトシート" 的見た目で、現状の「カード形」とは情報量が大きく違う。

## 2. 仕入設定セクション (モック新規)

モックは `bg-slate-100` のグレーカードで以下を**読み取り専用**で表示:
- 仕入商品名 (`product.last_title`)
- 仕入商品URL (`product.source_url` をリンクで)
- 仕入サイト名 (`product.site`)
- 仕入価格 (`product.last_price`)
- 最終更新タイムスタンプ (`product.last_scraped_at` 相当)

現状: 同等情報は **基本設定アコーディオンの下部に小タイル3枚 + 内部仕入れメモアコーディオンの中** に散らばって配置されている。モックの「自動取得情報を一枚のカードにまとめて目立たせる」設計は無し。

> AGENTS.md の `Never expose source_url ... in the public catalog` 制約に注意。商品編集ページ (オーナー専用) ならソースURL表示は問題ないが、誤って公開カタログ側に出ない実装にする必要あり。

## 3. 商品説明セクション

| 観点 | モック | 現状 |
|---|---|---|
| 構成 | `テンプレート適用` 行 (青枠パネル) + 2列グリッド: `[商品名 JP]` + `[商品名 EN + 自動翻訳ボタン inline]` + `[説明 JP RTE]` + `[説明 EN RTE + 全文翻訳ボタン inline]` | テンプレート行 + JP 列と EN 列を**縦に**並べる (横ではない) |
| 翻訳ボタン | フィールドラベル横に `自動翻訳` (タイトル) / `全文翻訳` (本文) を**個別**で出す | `タイトル翻訳` `自動翻訳` を EN 列の冒頭に**まとめて**配置 |
| 英語欄ヒント | placeholder で促し | 「英語情報はまだ未入力です」コピー + `英語欄へ移動` ボタン (現状の方が親切) |
| RTE | 軽量 (Bold + List のみ表示) | TinyMCE 系で多機能 |

→ 並びが縦/横で変わるため、画面幅に対する情報密度が変わる。現状の「JP→EN を縦に並べて翻訳ボタンを集約」は親切モードに振っているが、モックの「並列で翻訳ボタンが各フィールドにある」方が量産時の編集導線は速い。

## 4. 販売設定セクション (← モックでの新名称)

モックの構成:
```
┌────────────────────────────────────────────────────┐
│ 販売設定                       [○ 個別設定を有効にする │ ← トグル
├────────────────────────────────────────────────────┤
│  販売価格 (¥21,500)        在庫数 (1)              │
│  「自動計算された価格が反映」  「※デフォルトは1」      │
│                                                    │
│  ─── (トグルONで現れる) ───                        │
│  ┌ 個別計算パラメータ ─────────────────────────┐ │
│  │  個別の利益率 (%)        個別の追加送料 (¥)  │ │
│  │                                              │ │
│  │                              [再計算ボタン]  │ │
│  └──────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────┘
```

現状: 同等機能は **バリエーション設定セクション内の `manual-price-override` ヘッダ** にあり、検索しないと辿りつきにくい。
- 販売価格は **バリエーション table の `v_price_*` 列** にあり、トグルとは別ブロック
- 在庫も同 table の `v_qty_*` 列
- モックは「単一バリアント」を前提とした 1価格 + 1在庫 の UI

差分の本質:
- **モックは単一バリアント前提**。販売価格/在庫を section 直下に並べる
- **現状はマルチバリアント前提**。table 行ごとに価格/在庫/SKU/重量/税が並ぶ

→ モック寄りに寄せるなら、(a)「バリアント1件のみのときは販売価格/在庫を section 直下に出す」フォールバック表示か、(b) variants をやめて単純化する大改修、のどちらかが要る。後者は既存のマルチバリアント仕様 (eBay CSV/Shopify CSV/在庫管理) を破壊するため非推奨。

## 5. SEO セクション (モック新規の独立化)

モックは独立カード。現状は「販売メモ・検索設定」アコーディオン内に Vendor/タグ/Handle/SEO Title/SEO Description が混在。

モックの追加要素:
- URLハンドル: `英語名と同期` バッジ + `readonly` で `psa10-one-piece-card` のように自動生成
- ページタイトル: `英語名と同期` バッジ + `readonly`
- 同期ロジック (JS): `title_en` の入力 → `seo_handle` (英数小文字+ハイフン化) と `seo_title` を更新

現状: 手動入力のみ。英語タイトルとの自動同期ロジックは **無い**。

## 6. 右サイドバー (商品ステータス + 分類)

モックの右側 1列:
- **商品ステータス** カード: `下書き / 公開中 / アーカイブ` の単一 select
- **分類** カード:
  - **カテゴリー** select (`トレーディングカード` / `フィギュア` ...)
  - **タグ** pill UI (Enter で追加)

現状: 同等の機能はあるが**配置が異なる**:
- 公開ステータス → `基本設定` アコーディオン内
- カテゴリー → **存在しない** (`Product` モデルに `category` 列なし)
- タグ → `販売メモ・検索設定` アコーディオン内 (PR #96 で pill UI 化済)
- 保存前チェック → 現状の右側に出るカード (モックには無い)

→ **カテゴリーフィールドはモデルから不在**。AGENTS.md の "Not yet implemented" にも `pricelist category filter` が挙がっているので、ここを実装すると公開カタログのカテゴリーフィルタとセットで新規機能になる。

## サマリ

PR #96 でカバーしたのは「タグ pill 化」「画像ライトボックス」「個別 margin/shipping override (UI トグル + プレビュー API + 永続化)」の **既存ページに最小侵襲で機能を追加する変更**だけです。
モックが要求しているのは **ページ全体の情報設計を組み替える大規模リファクタ** で、たとえば:

- 全 `<details>` accordion を廃止しフラットなカード型セクションへ
- 画像グリッドを 10列極小サムネ + 円形 hover アクションへ
- 「仕入設定」「販売設定」「SEO」の独立カードを新設
- 右サイドカラム (ステータス + 分類 + タグ) を新設し、現状の「保存前チェック」を退ける
- 単一バリアント前提の `販売価格 + 在庫数` を section 直下に出す (マルチバリアントとの両立 UX を要決定)
- `英語タイトル → URLハンドル/ページタイトル` の自動同期 JS
- `category` カラム新設 + マスタとなる選択肢の管理 UI
- ヘッダ右に保存ボタン (sticky)

…まで実装する必要があります。
