# 実施不能理由
- 問題文内の対象環境、対象機能、権限体系、外部連携先がプレースホルダのままで、実運用環境固有の疎通先が未提示。
- ローカル検証では単一ブラウザ・単一ユーザー中心となり、マルチロール・マルチ端末の完全再現ができない。
- 外部サイト連携は本番影響・対向制限の観点から無断で実行すべきでない。

# 代替確認方法
- `tests/test_e2e_routes.py` と fixture ベースの smoke で回帰の土台を確保する。
- 公開データ露出、CSRF、直接URLはローカル UI とコード読解で補完する。
- Redis/RQ は `tests/test_rq_scrape_e2e.py` のスキップ条件と CLI runbook で事前確認し、別途環境を整えて再実行する。

# 残存リスク
- 本番URL配下の reverse proxy / cookie secure / CSP / HTTPS 特有の不具合
- 外部サイト仕様変更によるスクレイプ失敗
- 実機ブラウザ差異による入力・レイアウト問題
- マルチテナントでの IDOR / 権限制御漏れ
- 大量商品時の一覧/公開カタログ性能劣化

# 追加で必要な権限 / データ / 環境
- 対象ステージングまたは本番相当 URL
- 管理者 / 一般ユーザー / 閲覧専用など複数ロールのテストアカウント
- Redis/RQ を含む split 構成の検証環境
- 外部連携のモックまたはサンドボックス資格情報
- 大量データセット、期限切れトークン、削除済みデータなど境界ケース用データ
