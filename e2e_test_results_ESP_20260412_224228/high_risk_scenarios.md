# 高リスクシナリオ最小セット
| 優先 | シナリオ | 関連ケース | リリース判断 |
|---|---|---|---|
| 1 | 新規登録→ログイン→商品手動登録→一覧/詳細整合 | TC-E2E-001, 004, 021 | 失敗ならリリース不可 |
| 2 | 商品更新→価格表作成→公開URL表示→内部情報非露出 | TC-E2E-005, 007, 008, 028 | 失敗ならリリース不可 |
| 3 | エクスポートCSV生成 | TC-E2E-009 | 失敗なら主要業務停止 |
| 4 | 未ログイン/ログアウト後の保護画面遮断 | TC-E2E-027, 030 | 失敗ならセキュリティ事故 |
| 5 | スモークCLI完走 | TC-E2E-040 | 失敗なら運用ゲート無効 |

# リリース可否判断に直結する導線
- 商品を登録・更新して販売用データへ反映できること
- 公開価格表が必要な情報のみを露出して外部共有できること
- CSV エクスポートが壊れていないこと
- 認証境界が破綻していないこと
- リリース後のスモーク/監視導線が完走すること

# クリティカルパス
1. `/register` または `/login`
2. `/shops`
3. `/products/manual-add`
4. `/product/<id>`
5. `/pricelists/create` → `/pricelists/<id>/items`
6. `/catalog/<token>`
7. `/export/shopify`

# データ破壊 / 金銭影響フロー
- 販売価格更新後に古い価格が CSV / 公開画面へ残る
- 在庫やステータス変更がエクスポートへ反映されない
- 二重送信で商品や価格表が重複生成される
- 削除/アーカイブ商品が公開面やエクスポートへ残存する

# 権限事故フロー
- 未ログインで保護画面へ直接アクセスできる
- ログアウト後の戻るで商品データが見える
- 公開カタログで内部 sourcing 情報が見える
- 他ユーザーID推測で詳細閲覧/更新できる

# 外部連携失敗時の確認項目
- スクレイプ失敗時にユーザーへ失敗理由が分かるか
- ジョブ状態 API が stuck / timeout / failed を正しく返すか
- CSV 生成失敗時に空ファイルや 500 を返さないか
- 画像URL不正時に画面崩れしないか
- Redis/RQ 未接続時のガイダンスが適切か
