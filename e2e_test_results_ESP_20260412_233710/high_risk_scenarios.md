## 高リスクシナリオの最小実行セット
| 優先 | シナリオ | 今回結果 | リリース判断への意味 |
|---:|---|---|---|
| 1 | 登録 → ログイン → ログアウト → 戻る | Pass | セッション制御の最低限は成立 |
| 2 | ユーザーAで 商品作成 → 更新 → 一覧反映 → DB反映 | Pass | 保存成功と実データ一致を裏取り |
| 3 | ユーザーBで A の商品/価格表/API に直アクセス | Pass | 主要IDORは今回範囲で未検出 |
| 4 | 価格表作成 → 商品追加 → 公開カタログ → analytics | Pass | 主要公開導線は成立 |
| 5 | notes に script を含めて公開 | Pass | notes のサニタイズは機能 |
| 6 | 外部CDN不達で編集画面を開く | Fail | 編集系画面のレジリエンスに懸念 |
| 7 | 為替API不達で公開カタログを開く | Pass | フォールバックレートで継続利用可 |

## リリース可否判断に直結する導線
- 認証とレート制限
- 商品の登録/編集/一覧反映
- 公開価格表の表示と内部情報非露出
- 他ユーザー資産への直接アクセス拒否
- 編集UIが外部依存障害にどこまで耐えるか

## クリティカルパス
1. ユーザー登録
2. ショップ登録
3. 商品手動追加
4. 商品編集（価格/在庫/ENタイトル）
5. 価格表作成
6. 商品追加
7. 公開カタログ共有
8. 閲覧記録の集計

## データ破壊や金銭影響のあるフロー
- 商品 variant 価格更新と `selling_price` 代表値同期
- 価格表 custom_price 設定と公開価格の差異
- 価格ルールの更新による将来の一括再計算
- CSVエクスポート未確認のため、販売価格/在庫の downstream 反映は残リスク

## 権限事故の恐れがあるフロー
- `/product/<id>` 直接アクセス
- `/pricelists/<id>/items` / `/analytics` / `/edit` 直アクセス
- `/api/products/<id>/inline-update` PATCH
- 公開トークンURLと管理URLの混同

## 外部連携失敗時の確認項目
| 項目 | 今回観測 | 補足 |
|---|---|---|
| TinyMCE CDN | Fail | 編集系3画面で JS 例外 |
| Chart.js CDN | 劣化だが継続 | analytics の表は表示、可視化は欠落し得る |
| open.er-api.com | Pass | fallback rate 表示で継続 |
| 外部画像/フォント | 劣化 | example.com / Google Fonts はブロックされた |
| scrape 外部サイト | 未実施 | 安全条件不足 |

## 優先実行理由
- リリース直後に顧客・運用担当が必ず通る導線であり、壊れると商品運用・公開共有が停止するため。
- 金額・在庫・公開URL・権限は事後修復コストが大きいため。

## 未実施時の残存リスク
- CSV入出力で価格/在庫が誤って出力される可能性。
- archive/trash 状態遷移で公開面や一覧と不整合になる可能性。
- スクレイピング失敗が tracker/UI/API で正しく見えない可能性。
- モバイル/小画面で操作不能になる可能性。
- CDN/CSP/社内ネットワーク条件による編集UX劣化が本番で顕在化する可能性。
