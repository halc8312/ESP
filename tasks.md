# 今後の開発計画

このドキュメントは、現在のESPプロジェクトにおける未実装機能および将来的な機能拡張に向けたロードマップです。

## フェーズ1: 複数バリエーション対応 (基本実装完了)
データベース構造の改修とCSV出力ロジックの対応は完了しました。

### 完了済みタスク
- [x] **DBスキーマの再設計**
    - `Variant` テーブルを新規作成 (`id`, `product_id`, `option1_name`, `option1_value`, `price`, `sku`, `inventory`, etc.)
- [x] **UI/UXの改修 (基本)**
    - 商品詳細画面に「バリエーション編集」セクションを実装（編集・保存のみ）。
- [x] **CSVエクスポートロジックの修正**
    - `export_shopify`: バリエーションごとに行を出力するロジックへ変更。
    - `export_stock/price_update`: バリエーションSKU (Handle/SKU) に基づく出力へ変更。
- [x] **バリエーション行の追加・削除機能**
    - UIから自由に行を追加・削除できるJavaScript機能を実装。

### 残タスク (UI強化)
- [x] **バリエーション一括生成ウィザード** (Optional)
    - 「S, M, L」などを入力して一括生成する機能。

## フェーズ2: 認証とセキュリティ (完了)
本番運用に向けたセキュリティ対策を導入しました。

### 完了済みタスク
- [x] **ログイン機能の実装**
    - `Flask-Login` を用いたユーザー認証。
    - 管理ユーザー作成コマンド (`flask create-user`) の実装。
- [ ] **操作ログの記録** (Optional)

## フェーズ3: 運用効率化と機能拡張 (Priority: Low)
日々の運用をよりスムーズにするための機能群です。

### タスク
- [ ] **画像編集機能**
    - ツール上で画像のトリミングや回転を行えるようにする。
- [ ] **一括編集機能**
    - 商品一覧画面で、チェックした商品のステータスや価格を一括で変更する機能。
- [ ] **eBay連携の強化**
